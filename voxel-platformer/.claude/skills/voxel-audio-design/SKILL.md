# Voxel audio design

All game audio is **synthesized from code** — there are no recorded asset
files. `scripts/generate-audio-samples.mjs` builds every WAV from additive/
noise/filter primitives and writes them to `public/audio/8bit/` as **22050 Hz,
8-bit, mono PCM**. The build is **deterministic and byte-locked**: a test
pins each file's size + SHA-256, so touching the synth means regenerating and
re-locking the hashes.

This skill covers two jobs:

1. **Authoring a sound** — a one-shot SFX (combat hit, pickup, UI cue) or a
   looping bed (music, weather, liquid, fire).
2. **Wiring it to gameplay** — register the asset, then play it from a system
   callback (`audio.play` / `playSpatial` / `playMusic`) or a level/script.

For depth beyond the quick path, read the matching file in `reference/`.

---

## The pipeline (every audio change follows this)

1. **Compose** a generator in `scripts/generate-audio-samples.mjs` and add a
   `writeWav('my-sound.wav', myGenerator())` call. Reuse the existing synth
   primitives (`note`, `pluck`, `chirp`, `noiseBurst`, `filteredNoise`,
   `crackle`, `rumble`, `kick`, `pianoNote`, `padNote`, `waterDrop`, …). Add a
   new primitive only when no combination of existing ones fits.
2. **Generate**: `npm run audio:samples` (or `node scripts/generate-audio-samples.mjs`).
   This rewrites `public/audio/8bit/`.
3. **Register** in `src/game/audio.ts`: add a `GameAudio` id constant **and** a
   manifest entry under the right group — `sounds` (sfx bus), `music`, or
   `stingers`. Use `path('my-sound.wav')`.
4. **Re-lock the hash**: add/update the file's `{ bytes, sha256 }` in
   `tests/audio-assets.test.ts`. Get the values with:
   ```bash
   stat -c%s public/audio/8bit/my-sound.wav
   sha256sum public/audio/8bit/my-sound.wav
   ```
5. **Play it** — wire a system callback in `src/client.ts`, or set it as a
   level `environment` bed, or call it from a script. See
   `reference/wiring-and-registration.md`.
6. **Validate**: `npm test` (the audio-assets test enforces the hashes) and
   `npm run typecheck`.

## Authoring a one-shot SFX

Start from the closest existing recipe in the script (e.g. `melee-hit.wav`,
`shield-block.wav`, `bolt-hit.wav`). The shape is always:

```js
writeWav('my-hit.wav', mix(0.30, [          // mix(DURATION_seconds, layers)
    kick(0.00, 0.30),                        // body thump
    noiseBurst(0.00, 0.05, 0.40, 8151),      // transient (seed = unique int)
    filteredNoise(0.01, 0.12, 0.10, 1400, 8161), // grit
]))
```

Key rules:

- **`mix(duration, layers)` — the first arg is DURATION, not gain.** All
  loudness lives in each layer's `amp` argument.
- **Determinism is mandatory.** The synth must never call `Math.random()` or
  `Date.now()` — they throw at generation time and would break the hash. Every
  noise primitive takes a `seed` integer; give each layer (and each file) a
  distinct seed. For per-call variety at *play* time, jitter `rate` in the
  **client callback**, not the synth.
- **8-bit is shallow.** `signalToU8` clamps to [-1, 1] and scales by 100 around
  128. Keep summed peaks under ~1; a little transient clipping reads as crunch,
  gross clipping reads as broken. Combat/rapid SFX stay **short** and get a
  modest `maxInstances` so a flurry doesn't fatigue or starve voices.

Recipe vocabulary (which primitive for which sound) is in
`reference/synth-cookbook.md`.

## Authoring a looping bed (music / ambience)

Loop generators return a `Float32Array` (via `make(duration)` + layered
primitives) and **must end with `crossfadeEnds(signal, fadeS)`** so the loop
point doesn't click. Music uses `pianoNote` (acoustic-ish voice) over
`padNote` (warm bed); keep it sparse, low-amplitude, and long so the repeat is
hard to catch. Register looping beds in the **`music`** group (`loop: true`).

- Per-location music: set the level's `environment: { soundId, volume }`.
  **`environment.volume` overrides the asset's manifest volume** for that bed.
- Script-triggered music: `audio.play('music.<id>', { fade: 1.5 })`
  cross-fades on the music bus.

The C418-style ambient set (`amb-start/garden/town/tension/cave`) and the
`waterDrop` cave voice are worked examples of this path.

## Validation

```bash
npm run audio:samples   # regenerate WAVs after any synth change
npm test                # audio-assets.test.ts enforces byte size + SHA-256
npm run typecheck       # after touching audio.ts / systems / client.ts
```

If `audio-assets.test.ts` fails with "hash changed" or "byte size changed",
you edited the synth without re-locking the hashes (step 4) — that's the
contract, not a flake.
